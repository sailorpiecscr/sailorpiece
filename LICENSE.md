## 🛠️ Troubleshooting Common Issues

**❌ Script GUI doesn't appear after execution**
The overlay may be blocked by a Roblox update changing UI layer priorities.
**Solution:** Re-inject your executor after fully closing and reopening Roblox; don't rely on executor "re-attach" if the game was updated.

**❌ Kill Aura not targeting enemies**
The target detection radius may be misconfigured, or the module toggled off by default on fresh load.
**Solution:** Open the config block and confirm `KillAura.Enabled = true` and `KillAura.Range` is set to `40` or higher before executing the **Sailor Piece Script**.

**❌ Boss Rush teleport loop freezing mid-cycle**
High server latency or a boss that despawned before teleport completed can break the chain.
**Solution:** Increase `BossRush.RespawnDelay` to `5` in the config — this gives the server more time to register the spawn before the teleport fires.

**❌ Android executor fails to inject**
Some Android versions block executor `.apk` memory access permissions after OS updates.
**Solution:** Go to `Settings > Apps > [Executor Name] > Permissions` and manually re-enable all requested permissions, then cold-relaunch both the executor and Roblox.

**❌ Discord webhook not receiving logs**
The webhook URL field is empty or has an extra space in the config.
**Solution:** Re-paste your webhook URL directly into `Webhook.URL = ""` in the config with no trailing spaces; test with `LogBossKills = true` first.

**❌ Safe Mode not reducing detection risk**
Safe Mode requires `SafeMode.Enabled = true` AND `ActionDelay` above `0.2` to be meaningful.
**Solution:** Set `SafeMode.ActionDelay = 0.35` for a comfortable balance between speed and natural timing in the **Roblox Sailor Piece Script Pastebin** config.
